# Guia de Contribuição

Obrigado por considerar contribuir para o projeto Controle de Treinamento! Este documento contém diretrizes para ajudá-lo a contribuir de forma eficaz.

## Código de Conduta

Ao participar deste projeto, você concorda em seguir nosso Código de Conduta, que promove um ambiente acolhedor e respeitoso para todos os colaboradores.

## Como Contribuir

### Reportando Bugs

Antes de reportar um bug, verifique se ele já não foi reportado na seção de issues. Ao reportar um novo bug, inclua:

- Versão do Python e Django utilizados
- Passos para reproduzir o problema
- Comportamento esperado vs. comportamento observado
- Mensagens de erro completas, se houver
- Informações do ambiente (sistema operacional, navegador, etc.)

### Sugerindo Melhorias

Você pode sugerir melhorias através da seção de issues. Descreva claramente:

- A melhoria proposta
- O problema que ela resolve
- Exemplos de uso, se aplicável

### Contribuindo com Código

1. Faça um fork do projeto
2. Crie uma branch para sua feature (`git checkout -b feature/nova-feature`)
3. Commit suas mudanças (`git commit -m 'Adiciona nova feature'`)
4. Push para a branch (`git push origin feature/nova-feature`)
5. Abra um Pull Request

## Padrões de Código

### Python

- Siga o PEP 8 para estilo de código
- Utilize docstrings para funções, classes e módulos
- Mantenha mensagens de commit claras e descritivas
- Escreva testes para novas funcionalidades

### Django

- Siga as convenções do Django para estrutura de projetos
- Utilize os recursos de validação do Django Forms
- Siga as práticas recomendadas de segurança do Django

### Frontend

- Utilize HTML5 semântico
- Siga as convenções do Bootstrap 5
- Mantenha o CSS organizado e comentado

## Processo de Pull Request

1. Certifique-se de que seu código segue os padrões do projeto
2. Execute os testes existentes e adicione novos testes quando apropriado
3. Atualize a documentação, se necessário
4. Descreva claramente as mudanças no PR
5. Solicite revisão de outros colaboradores

## Ambiente de Desenvolvimento

Consulte o README.md para instruções sobre como configurar o ambiente de desenvolvimento.

## Testes

- Escreva testes unitários para novas funcionalidades
- Execute todos os testes antes de enviar um PR
- Mantenha uma cobertura de teste adequada

## Documentação

- Atualize a documentação quando modificar funcionalidades existentes
- Adicione documentação para novas features
- Mantenha a documentação clara e concisa

## Dúvidas?

Se tiver dúvidas sobre como contribuir, abra uma issue perguntando ou entre em contato com os mantenedores do projeto.
